WHAT THIS IS
An opinionated, linear review pipeline for Claude Code. A feature enters one end
and gets beaten up by 8 personas (survey → product-review → design-review →
arch-review → fresh-eyes → implement → code-review → qa). Each skill is a
self-contained SKILL.md. No runtime deps beyond Claude Code, git, and
playwright-cli for QA.

STACK
Pure markdown. No code, no framework, no dependencies.
Skills are Claude Code SKILL.md files with YAML frontmatter.
QA skill depends on @playwright/cli (npm).

CODEBASE HEALTH
- Structure: GOOD — flat skill directories, one SKILL.md each, clean separation
- Test coverage: N/A — no code to test, though the skills themselves are untested
- Documentation: ADEQUATE — README covers the pipeline, install, principles
- Dependency freshness: N/A — no manifest
- Git hygiene: EMPTY — zero commits, all files untracked, .DS_Store and .swp in tree

OPEN WOUNDS
- Zero commits. Nothing is versioned. One rm -rf and it's gone.
- .DS_Store and .README.md.swp are sitting in the root untracked.
  No .gitignore exists.
- The skills are not installed. They live in skills-from-claude-ai/ but
  Claude Code won't discover them there. They need to be in
  ~/.claude/skills/gauntlette/ (or symlinked) to register as /slash-commands.
- README install says `playwright-cli install-skill` — the actual command
  is `playwright-cli install --skills`.

None. Project is too new for debt. It has "hasn't started yet" problems, not
tech debt problems.
